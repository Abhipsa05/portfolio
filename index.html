Experience

May 2024 — Present

ShareChat

Software Development Engineer — Platform & DevOps · Bangalore

Nexus is ShareChat's in-house ambient service mesh, built on Envoy and etcd v3. It handles service discovery, traffic management, health checking, and dynamic configuration for 120+ microservices at 7.5M requests/second across the ShareChat and Moj production environments — and I own it end to end.

  • Build and operate the mesh control plane, which watches Kubernetes pod events, registers endpoints in etcd, and generates and pushes xDS configuration to a fleet of Envoy proxies in near-real-time — every route and endpoint in production flows through this pipeline.
  • Run the mesh in ambient mode using eBPF: programs injected at the node level make pods mesh-members without sidecar restarts, with dedicated waypoint proxies handling egress — improving observability and runtime performance at scale.
  • Achieved a 95% reduction in Network Inter-Zone Egress (NIZE) costs through zone-aware routing — keeping traffic inside the availability zone unless health or capacity demands otherwise.
  • Cut CPU and memory usage by 50% across critical mesh components (control plane, registration, and health-checking) through performance optimization.
  • Built a metric-based load balancing system that feeds live utilization signals into routing decisions, improving service availability while minimizing cost.
  • Own the distributed health-checking system that continuously validates endpoints and updates their state in etcd, so traffic never reaches an unhealthy pod.
